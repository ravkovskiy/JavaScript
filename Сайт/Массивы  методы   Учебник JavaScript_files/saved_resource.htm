                                                                                      Потому что, во-первых, когда obj === null цикл while не выполнится и соответственно не выполнится присваивание obj = null. Ну а, во-вторых, сам исходный list не меняется при присваивании obj т.к. каждый раз в obj просто присваивается ссылка на новый объект obj.next. Это аналогично ситуации если бы мыс делали так:

                                                                                      var obj = list;
obj = list.next;
alert(obj.value); //2
alert(list.value); //1


                                                                                      Как видно сам объект list не изменился при присваивании obj. Потому что в переменной obj лежит ссылка на объект list. При присваивании переменной obj нового значения эта ссылка просто перезаписывается на это значение. В данном примере в obj попадает ссылка на объект list.next. В переменно list так же хранится ссылка на этот объект list. От того, что в obj теперь ссылка на другой объект переменной list ни жарко ни холодно т.к. в ней то лежи старая ссылка на объект. Поэтому присваивание нового значения obj на содержании самого объекта никак не отражается. Это применимо и в обратную сторону:

                                                                                      //заносим в obj ссылку на объект list
var obj = list;
//в переменную list заносим ссылку на объект list.next
list = list.next; 
alert(obj.value); //1
alert(list.value); //2


                                                                                      Видно что изменение list не повлияло на obj, потому что переменные ссылаются не друг на друга, а на объекты в памяти. Поэтому перезапись самих переменных не влияет на другие ссылки.
                                                                                      Но, если бы мы через ссылающуюся переменную изменили бы свойство объекта:

                                                                                      var obj = list;
obj.value = 111; //меняем свойство
alert(obj.value); //111
alert(list.value); //111


                                                                                      то изменение отражается в обоих переменных. Это происходит потому, что при изменении obj.value мы меняем свойство value у объекта на который ссылается obj. А она ссылается на тот же объект что и list. Т.е. при изменеии свойства объекта мы меняем само свойство. А при изменении переменной ссылающейся на объект мы меняем содержимое только этой переменной, а не объекта.

Для наглядности это можно было бы записать в 2 строки:
                                                                                                                                                arr.splice(i, 1);
                                                                                                                                                i--;
