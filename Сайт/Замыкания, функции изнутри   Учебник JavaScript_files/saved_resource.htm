            Ключевое слово "function" служит для создания функции. Например, в выражении
            var f = function(){}
            происходит следующее:
            1) Создается новая полноценная функция.
            2) Переменной f присваивается ссылка на эту функцию.
            С return function(){} происходит тоже самое, только ссылка передается не в переменную, а возвращается из функции.

                                                                                                                                Я уже кажется разъяснял чуть ниже как работает этот самый Лексикал Энвайренмент :) В данном примере есть функция f(), в которой рандомится некоторое число и присваивается в переменную а. Сама функция f возвращает другую анонимную функцию. При каждом запуске функции f будет рандомиться новое число и присваиваться в новую переменную а. А возвращаемая функция будет иметь доступ к этой самой переменной а. Таким образом, если бы внутренняя возвращаемая функция имела код function() {alert(a);} то при обращении к любой из ячеек массива, вызывался бы алерт, который бы выводил СООТВЕТСТВУЮЩУЮ переменную а. То есть для каждой функции это была бы своя переменная а. В общем, механизм следующий. Запускается функция f. Она возвращает внутреннюю функцию, которая присваивается в ячейку массива. Эта функция имеет доступ ко всем переменным, которые находились в момент вызова функции f() внутри нее. Таким образом, эти переменные не стираются, а сохраняются в памяти, пока существует эта возвращенная функция. Таким образом и получаются независимые счетчики. Отсюда же и фраза - хранить в замыкании. Пока к переменным что-то обращается - они существуют. Удалите внутреннюю функцию, которая к ним обращаются - очистится и память, которая эти переменные хранит. Достаточно сложная для объяснения тема, но если всечь, то всё встанет на свои места :)
