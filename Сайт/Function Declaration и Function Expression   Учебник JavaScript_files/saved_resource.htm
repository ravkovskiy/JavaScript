                В вашем примере вы как раз используете function expression.
                Когда вы берете в свою функцию скобки, то она становится выражением, а это уже не function declaration.

                Если вы будете использовать свою функцию как function declaration и уберете скобки вокруг нее, то вызов ее на месте даст ошибку. На стадии инициализации кода функция будет обработана, а скобки вызова останутся. При выполнении кода интерпритатор наткнется на них, не распознает и выдаст error. Поэтому function declaration вызвать на месте технически не возможно.

                В if declaration запихнуть конечно можно, но код выполнится не так как вам бы этого хотелось. Читайте материал внимательнее.

                                                    Zuev Алексей • месяц назад

                                                    Смотри, function declaration это просто обычная функция которую ты пишешь в коде

                                                    function foo() {
                                                        alert('Привет мир');
                                                    }


                                                    Если интерпритатор при инициализации кода увидел function то сразу понимает, что это function declaration.

                                                    function expression это точно такая же функция - все то же самое.

                                                    Но если ты ее не просто написал как в примере выше, а усложнил ее... например присвоил ее в переменную

                                                    var x = function foo() {
                                                        alert('Привет мир');
                                                    }

                                                    или взял в скобки

                                                    (function foo() {
                                                        alert('Привет мир');
                                                    })

                                                    или создал ее как элемент массива или метод объекта

                                                    var arr = ['str' , 5, true, function foo() {alert('Привет мир');}  ]

                                                    то этими действиями ты сделал из нее function expression.

                                                    Интерпритатор в этом случае сразу не увидит function он увидит в первую очередь создание переменной x, какое-то выражение в скобках, создание массива arr, но не увидит нашу функцию, а оставит ее на потом и разберет уже при выполнении.

                                                    function declaration интерпритатор видит сразу, в отличии от expression. В этом вся и разница.

                                                    Самое важное - что function declaration он обрабатывает в ПЕРВУЮ ОЧЕРЕДЬ при инициализации кода, а function expression, так как он не увидел их сразу - обрабатывает уже по мере выполнения кода, когда до них дойдет очередь. Это самое главное что необходимо понять из этого урока. Потом при изучении глобального объекта вы с этим столкнетесь и поймете это все еще более глубже.

                                                                                                Все просто: переменная scareMe является внешней переменной для основной функции, и в то же время указывает на саму функцию. Когда же внутри функции мы присваиваем scareMe другую функцию, то просто ссылка в переменной scareMe начинает указывать уже на функцию, которая возвращает ("Double boo"), а не на исходную функцию, которая "пропадает" после того, как на нее перестала указывать переменная.

                                                                                                Если же поставить оператор var внутри основной функции перед scareMe, то фокуса не произойдет, потому что изменяться будет уже внутренняя переменная функции, а внешняя, имеющая такое же имя все также будет указывать на исходную функцию без изменений.
